# 开发约束文档

## 1. 错误处理原则
- ❌ 不允许使用 try-except 吞掉错误后不做处理
- ❌ 不允许使用 `pass` 或空 `except` 块
- ✅ 所有异常必须记录日志（logging），包含完整的 traceback
- ✅ 数据层异常向上层抛出，由 UI 层统一展示给用户
- ✅ 遇到问题流程：重现问题 → 分析问题 → 定位根因 → 修复问题

## 2. 开发流程
- 开发新功能前：先设计 → 再规划 → 再实施
- 修改 Bug 前：先编写复现步骤 → 定位根因 → 修复 → 验证不影响已有功能
- 每次修改后必须运行测试验证

## 3. 架构约束
- 严格遵循三层架构：UI 层 → Service 层 → API/Data 层
- 层与层之间通过接口/抽象进行通信，不得跨层调用
- 改动一层不应影响其他层
- 修改 Bug 不能影响原有功能

## 4. 文档约束
- 所有过程记录到 `docs/changelog.md`
- 架构变更记录到 `docs/architecture.md`
- 本文档保持最新，不得删除

## 5. 代码规范
- 使用 type hints
- 每个函数必须有 docstring
- 常量和配置集中管理在 `src/config.py`
- 不使用魔法数字

## 6. 稳定性要求
- 所有外部 API 调用必须有超时设置
- API 错误必须有合理的降级策略（如返回空数据 + 日志警告，而非崩溃）
- 使用 logging 而非 print

## 2. 模块职责与接口（概要签名）

### 2.1 datasource/interfaces.py（端口抽象）
```python
class EtfBasicSource(Protocol):
    def list_etfs(self) -> list[EtfBasic]: ...
    def get_etf(self, code: str) -> EtfBasic | None: ...

class ValuationSource(Protocol):
    def get_latest(self, index_code: str) -> IndexValuation | None: ...
    def get_history(self, index_code: str, start: date, end: date) -> list[IndexValuationPoint]: ...

class ConstituentSource(Protocol):
    def get_constituents(self, index_code: str) -> list[ConstituentStock]: ...
```
> 说明：用 `typing.Protocol` 定义端口；具体实现可替换，services 仅依赖此。

### 2.2 services（业务层，依赖接口）
| 类/函数 | 输入 | 输出 | 说明 |
| --- | --- | --- | --- |
| `SearchService.search(keyword, page, size)` | str,int,int | `ETFSearchResult[]` | FR-01 检索 |
| `ValuationService.get_valuation(code)` | str | `ValuationView` | FR-03 估值（含 applicable 判定） |
| `PercentileService.get_percentile(code, window)` | str,Window | `PercentileView` | FR-04 分位 |

## 3. 数据结构设计（ER 概要）
实体：`EtfBasic`(1) ——跟踪——▶ `IndexValuation`(1)、`IndexValuationHistory`(N)、`IndexConstituent`(N)
外键：`track_index_code = index_code`

表字段定义见 SRS 第 4 节 DR-01~05（etf_basic / index_valuation / index_valuation_history / index_constituent）。

---

## 4. 外部接口设计（概要）
- **入站**：REST JSON，base `/api`；错误统一 `{ "code": int, "message": str }`。
- **出站（数据源）**：遵循 `interfaces.py`；实现见详细设计。
- **定时**：`scheduler/jobs.py` 提供 `refresh_etf_basic()`、`refresh_valuation()`、`refresh_constituents()`。

---

## 5. 模块间协作（关键流程）
**检索流程**：`api → SearchService → EtfRepo.search_basic`（库内模糊匹配，命中缓存）
**估值流程**：`api → ValuationService → EtfRepo.get_basic(取index_code) → ValuationRepo.get_latest`（无则经 datasource 拉取并回写）
**分位流程**：`ValuationService/PercentileService → ValuationRepo.get_history → 分位算法`
**成分股流程**：`ConstituentService → EtfRepo(取index_code) → ConstituentRepo.get_constituents`

---

## 6. 松耦合要点（概要）
- services 不 `import` 任何具体数据源；只 `import datasource.interfaces`。
- 所有具体实现经 `core/di.py` 注入；测试用 Mock 实现替换。
- 数据库仅经 repositories；切换 SQLite/PG 不动 services。
- 新增数据源 = 新增一个实现 `interfaces` 的类 + 在 `di.py` 注册，services/api 零改动。

---

## 7. 非功能设计映射
- 性能：repositories 命中缓存；历史接口分页。
- 可用：ValuationService 内置"主源→兜底源"选择（详细设计展开）。
- 合规：schemas 出参统一带 `source` 与 `update_time` 字段。

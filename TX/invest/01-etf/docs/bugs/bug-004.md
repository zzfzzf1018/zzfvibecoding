# BUG-004: 点击 562550 绿电ETF 后控制台大量报错、成分股空白

- 关联任务：docs/tasks/task-02.md
- 状态：已修复
- 发现时间：2026-07-11
- 影响文件：`src/app/datasource/_akshare_bridge.py`、`src/app/datasource/akshare_src.py`

## 现象（BUG 描述）

浏览器点击 `562550 绿电ETF华夏` 后：
1. 控制台刷屏一堆 WARNING，包括：
   - `akshare.index_code_id_map_em 第N次失败：ProxyError ... Unable to connect to proxy`
   - `akshare.stock_index_pe_lg ... KeyError: '中证绿色电力指数收益率'`
   - `akshare.index_stock_cons_csindex ... ValueError: Excel file format cannot be determined`
   - `akshare.stock_zh_index_value_csindex ... UnicodeEncodeError: 'ascii' codec can't encode characters ...`
2. 「成分股」区块没有任何数据。

## 根因（Root Cause，多层叠加）

### R1 跟踪指数名提取错误（核心数据 bug）
`fund_overview_em` 的「业绩比较基准」列对绿电ETF 的值为 `中证绿色电力指数收益率`
（这是基准收益口径，真实跟踪指数是 `中证绿色电力指数`）。代码把该值原样当作
`track_index`，导致：
- 后续名->码解析、lg 估值都用了错误的名字；
- `stock_index_pe_lg` 文档明确只支持 12 个主流宽基（`symbol_map` 不含该指数），
  即使名字正确也会 `KeyError`，而传错名字更必然 `KeyError`。

### R2 指数代码解析不到（代理导致运行时映射失败）
`track_index_code` 由 `_resolve_index_code(track_name)` 解析：
- `track_name="中证绿色电力指数收益率"` 不匹配内置 `_COMMON_INDEX_MAP`；
- 运行时映射 `_runtime_index_code_map()` 依赖 `index_code_id_map_em`（eastmoney），
  而该环境配置了**不可达的 HTTP(S) 代理**，调用报 `ProxyError` 失败 → 缓存为空；
- 结果 `track_index_code = None`。

### R3 服务层把中文名当代码传给需代码的接口（放大报错）
`constituent_service` / `valuation_service` 用 `etf.track_index_code or etf.track_index`
作为 `idx`。当 `track_index_code=None` 时退回中文名，直接传给：
- `index_stock_cons_csindex(symbol)`：需要指数代码（拼 `.../{code}cons.xls` URL），
  传入中文名 → 拉到错误页 → `Excel file format cannot be determined`；
- `stock_zh_index_value_csindex(index_code)`：URL 含中文 → `UnicodeEncodeError`。

### R4 lg 对非支持指数无谓重试刷屏
`get_latest` / `get_history` 对任何指数都尝试 lg，对 12 个主流宽基之外的指数必
`KeyError` 且重试 3 次，产生大量噪音告警。

## 解决办法（均根治，未吞错）

### 修复 R2（根因——代理）
`src/app/datasource/_akshare_bridge.py`：子进程启动时清除不可达的代理环境变量
（`HTTP_PROXY`/`HTTPS_PROXY`/`ALL_PROXY` 等）。该代理本身不可达，清除后改直连反而
可用；对已在 `NO_PROXY` 中的 sina / lg / csindex 等源无影响。从此 `index_code_id_map_em`
等 eastmoney 调用可正常拉取名->码映射。

### 修复 R1（数据规范化）
- 新增 `_strip_rate_suffix()`：剥离「收益率 / 全收益 / 全收益指数」后缀，
  `中证绿色电力指数收益率` → `中证绿色电力指数`；
- `AkShareEtfBasicSource.get_etf` 提取 `track_name` 后调用它，存入规范的 `track_index`。

### 修复 R3（代码校验 + 兜底映射）
- 新增 `_is_index_code(s)`：判断是否为 6 位数字（可选 `.SH/.SZ/.CSI`）的指数代码；
- `_COMMON_INDEX_MAP` 增加 `"中证绿色电力指数": "931897"`（及别名 `"绿色电力"`），
  作为代理失败时的确定性兜底；
- `AkShareConstituentSource.get_constituents`：`_is_index_code` 不通过直接返回 `[]`，
  不再用中文名调 csindex；
- `AkShareValuationSource.get_latest` 的中证兜底仅在 `_is_index_code(index_code)` 时调用，
  避免中文名触发的 `UnicodeEncodeError`。

### 修复 R4（lg 支持集）
- 新增 `_LG_SUPPORTED`（akshare `stock_index_pe_lg/pb_lg` 实际支持的 12 个宽基）；
- `get_latest` / `get_history` 仅对该集合内的指数调用 lg，其余直接跳过，消除 KeyError 刷屏。

## 影响范围

- 仅影响「跟踪指数非 12 个主流宽基、且运行时 eastmoney 映射拉不到代码」的 ETF（如各类
  行业/主题 ETF，562550 是典型）。此前这类 ETF 成分股空白、控制台刷 ProxyError/KeyError/
  Excel/Unicode 错误。
- 主流宽基（510300 等）行为不变：lg 估值/历史、csindex 成分股照常工作。
- 历史分位（pe/pb 历史）仍仅 lg 覆盖的 12 个宽基有数据；主题指数（如绿电）无历史源，
  优雅降级为空（不再刷错），属数据源覆盖限制而非 bug。
- 估值：主题指数走中证兜底，可拿到 PE + 股息率（中证无 PB 列，PB 为 None 属正常）。

## 验证

- 确定性单测（不联网）：`_strip_rate_suffix`、`_is_index_code`、`_resolve_index_code`、
  `_LG_SUPPORTED` 全部符合预期；`_resolve_index_code("中证绿色电力指数") == "931897"`。
- 真实联网：
  - `AkShareConstituentSource().get_constituents("931897")` → **count=50**（深圳能源 1.081%、
    川能动力 1.891%…），成分股正常出数，不再有 Excel 解析错误；
  - `AkShareValuationSource().get_latest("931897", "中证绿色电力指数")` →
    `pe_ttm=18.24, dividend_yield=2.3, source=akshare-csindex`，估值正常出数。
- `pytest` 全量 20 passed；改动文件 lint 零告警。

# bug-001：查询 ETF 后详情页"空模板"，无真实数据

- 记录人：AI（CodeBuddy）
- 日期：2026-07-11
- 关联记录：CHG-20260711-02（见 `docs/开发记录.md`）
- 严重级别：高（核心功能不可用）

## 1. BUG 描述
在 Web 页面搜索并点击某只 ETF（如 510300）后，详情面板只渲染"空模板"：
基本信息大量字段为空、估值 PE/PB 为空、历史分位显示"暂无历史分位数据"、
成分股显示"暂无成分股"。用户诉求：应**实际显示数据**。

## 2. 复现条件
- 环境：Windows + akshare 1.18.64（已安装）。
- 步骤：启动服务 → 搜索 510300 → 点击 → 观察详情面板。
- 输入：任意 ETF 代码。

## 3. 根本原因（Root Cause）
经逐源联网探测（见 `docs/tasks/task-02.md` 附录），定位为**数据源接口过期 + 数据从未落库**，非前端渲染问题：

| # | 根因 | 证据 |
|---|---|---|
| R1 | `AkShareEtfBasicSource.get_etf` 调用 akshare 已删除的 `fund_etf_info_em`，抛 `AttributeError` → 基本信息与 `track_index_code` 全空 | probe1/2：`module 'akshare' has no attribute 'fund_etf_info_em'` |
| R2 | `AkShareValuationSource.get_latest` 调用已删除的 `stock_index_value_name_em` → 无 PE/PB | probe1：`no attribute 'stock_index_value_name_em'` |
| R3 | `EmValuationSource.get_latest` 用东财 `f116/f117`（个股字段）取指数估值，返回垃圾值 `pe=5.5e13` | probe1：`pe_ttm=55441163127420.41` |
| R4 | `ConstituentService.get_constituents` 只读 `constituent_repo`，但**没有任何代码路径把数据源结果写入该 repo**（`refresh_constituents` 任务未被触发/未接入按需） | 代码审阅 + 空表 |
| R5 | `PercentileService.get_monthly_percentile` 读 `valuation_repo` 历史，但历史**从未被采集**（估值仅 append 当天 1 点） | 代码审阅 |
| R6 | CSIndex 估值兜底源 URL `…/valuation/indices` 返回 404；且按代码类接口（成分股/估值）需要指数**代码**，而 `fund_overview_em` 只给指数**名**，缺少名→码解析 | probe1(404)、probe6(名调用报错) |

## 4. 解决办法
- R1/R2：重写 `AkShareEtfBasicSource`/`AkShareValuationSource`，改用当前可用接口
  `fund_overview_em`（基本信息）、`stock_zh_index_value_csindex`（当前 PE/PB，按代码）、
  `stock_index_pe_lg`/`stock_index_pb_lg`（完整 PE/PB 历史，按指数名，支持主流宽基）。
- R3：为 `EmValuationSource` 增加取值合理性校验（0<PE/PB<1000），越界返回 `None` 走降级，避免污染缓存。
- R4：为 `ConstituentService` 注入 `constituent_sources`，`get_constituents` 在本地库为空时**按需拉取并落库**（`replace_constituents`）。
- R5：为 `PercentileService` 注入 `valuation_sources`，`get_monthly_percentile` 在历史不足时**按需拉取历史 PE/PB 并落库**（`append_history`），随后计算分位。
- R6：新增 `_resolve_index_code(name)`（内置主流指数名→码映射 + 运行时 `index_code_id_map_em` 增强），补齐 `track_index_code`，使成分股/中证估值按代码可用。

## 5. 影响范围
- 受影响：详情面板四个区块（基本信息 / 估值 / 历史分位 / 成分股）。
- 数据层：`etf_repo`（基本信息字段补全）、`constituent_repo`（首次按需写入）、`valuation_repo`（历史序列写入）。
- 接口契约：`ValuationSource` 协议 `get_latest`/`get_history` 增加可选 `index_name` 参数（向后兼容，默认 None）。
- 不破坏：旧 `get_percentile`/`PercentileView` 保留；前端渲染结构不变，仅补充数值格式化。
- 数据修复/迁移：无（首次访问自动按需采集并落库）。

## 6. 验证
- 起服务后访问 510300：基本信息含发行商/费率/规模；估值显示 PE/PB；历史分位 12 个月有数据；成分股列表有数据。
- `pytest` 全量通过；`docs/tasks/task-02.md` 附录记录各源探测结果。
- 已知限制：历史分位/估值的"按指数名"历史源（`stock_index_pe_lg`）仅覆盖主流宽基指数；
  行业/主题类 ETF 的分位可能"暂无"（已优雅降级，非崩溃）。成分股/中证估值按代码，依赖名→码映射命中。

# bug-002：估值/历史分位数据异常（同进程污染 + 解析 bug）

- 记录人：AI（CodeBuddy）
- 日期：2026-07-11
- 关联记录：bug-001（同一需求"详情面板显示真实数据"的深层根因）、CHG-20260711-02
- 严重级别：高（估值恒 1.0、历史分位无意义）

## 1. BUG 描述
按 bug-001 换用可用接口后，详情面板仍异常：
- 估值 PE/PB 恒为 `1.0`（或 `None`），与真实值（沪深300 约 PE 13.88 / PB 1.43）不符。
- 历史分位 `sample_count=1`、`degraded=True`，分位恒 `100%`，无参考意义（预期约 1327 个交易日样本）。

## 2. 复现条件
- 环境：Windows + akshare 1.18.64；uvicorn 常驻单进程。
- 步骤：启动服务 → 访问 510300 → 观察估值与历史分位区块。

## 3. 根本原因（Root Cause）
共三个独立根因：

| # | 根因 | 证据 |
|---|---|---|
| R1 | **akshare 同进程连续调用被限流/连接复用污染**：同一 Python 进程内第 2 次及以后的 akshare 网络调用返回脏数据（pe=1.0 / None），首次正常；两次调用间隔 6 秒则恢复。属数据提供方对同 IP 快速连续请求的限流，非 akshare 会话状态。uvicorn 常驻进程会持续触发。 | `_probeF/G`：第二次调用脏值；`_probeK`：加 6s 延迟恢复 |
| R2 | **`_to_float(_pick(...))` 传入列名而非行内值**：`_pick` 返回列名字符串（如"市盈率1"），`_to_float` 用正则从列名"市盈率1"中提取到"1" → 返回 `1.0`；列名"静态市盈率"无数字 → `None`。导致 `get_latest` 的 PE/DY 恒为 1.0。 | 代码审阅 + 修复后返回 13.88 |
| R3 | **`_to_date` 不识别子进程桥接的 ISO 8601 日期**：桥接用 `df.to_json(date_format="iso")` 把日期序列化为 `'2005-04-08T00:00:00.000'`，而 `_to_date` 仅支持 `%Y-%m-%d` 等格式，对所有行返回 `None` → `get_history` 的 `pe_map/pb_map` 均为空 → 返回 `[]` → 历史从未落库，分位 `sample_count=1`。 | `_probe_hist`：日期 raw `'2005-04-08T00:00:00.000'` → `_to_date` 返回 None；修复后 `len=1326` |

## 4. 解决办法
- R1：新增子进程隔离桥接 `src/app/datasource/_akshare_bridge.py` + 调度器 `src/app/datasource/_runner.py`。
  每次 akshare 调用在独立子进程执行（子进程内仅调用一次 = 永远"首次调用"），
  父进程经 `pd.read_json(io.StringIO(out), orient="split")` 还原 DataFrame；
  `run_df` 加全局最小调用间隔（`_MIN_GAP=1.5s`）+ 指数退避重试（`_MAX_RETRY=3`），
  失败按约束抛出/降级而非静默吞掉。`akshare_src.py` 全部改为 `run_df(...)`，删除 `import akshare`。
- R2：`get_latest`/`get_history` 统一改为 `_to_float(r[_pick(...)])` / `_to_date(r[_pick(...)])`（先用列名取行内值再解析），与 `get_history` 中 `r[pcol]` 写法一致。
- R3：增强 `_to_date`，优先用 `datetime.fromisoformat` 解析 ISO 8601（兼容 `T`/毫秒/时区），
  再回退 `%Y-%m-%d`/`%Y/%m/%d`/`%Y-%m-%d %H:%M:%S`/`%Y%m%d`；并过滤 `NaT/NaN` 等空值。

## 5. 影响范围
- 受影响：估值区块（PE/PB/股息率）、历史分位区块（月度序列与样本数）。
- 数据层：`valuation_repo` 历史序列首次可正常落库（约 1300+ 点/主流宽基）。
- 性能：所有 akshare 调用改为子进程执行并串行限速，单次详情首访耗时增加（数秒），
  但避免脏值；成分股逐股行业补全（`_get_industry`）会造成数百次子进程风暴，已禁用（`sw_l1=None`）。
- 不破坏：接口契约、前端结构不变。

## 6. 验证
- `get_latest("000300","沪深300指数")` → `pe_ttm=13.88, pb=1.43, source=akshare-lg`。
- `get_history("000300",...,"沪深300指数")` → `len=1326`，首点 2021-01-18 pe=16.03，末点 2026-07-10 pe=13.88。
- `get_monthly_percentile("510300", window_years=5)` → `sample_count=1327, degraded=False`，
  2026-07 月 `pe_percentile=76.5, pb_percentile=67.6`（真实合理）。
- 四区块（基本信息 / 估值 / 历史分位 / 成分股）均实际出数。

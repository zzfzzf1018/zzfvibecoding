# BUG-003: `run.ps1 -Seed` 启动时 `fund_etf_category_sina` 报错

- 关联任务：docs/tasks/task-02.md
- 状态：已修复
- 发现时间：2026-07-11
- 影响文件：`src/app/datasource/akshare_src.py`

## 现象（BUG 描述）

使用 `powershell -ExecutionPolicy Bypass -File .\scripts\run.ps1 -Seed` 启动时，控制台在
"Seeding ETF basic list" 阶段反复出现 WARNING，且基础列表源刷新失败：

```
WARNING etf_query akshare.fund_etf_category_sina 第1次失败（将重试）: ...
File "...\akshare\fund\fund_etf_sina.py", line 40, in fund_etf_category_sina
    "node": fund_map[symbol],
TypeError: cannot use 'dict' as a dict key (unhashable type: 'dict')
...（重试 3 次后）...
WARNING etf_query basic source akshare 刷新失败: akshare.fund_etf_category_sina 子进程失败: ...
```

表象是 akshare 内部 `fund_map[symbol]` 抛 `TypeError`，但 akshare 代码本身无问题。

## 根因（Root Cause）

`AkShareEtfBasicSource.list_etfs` 调用子进程桥接时，把参数写成了位置参数且包装成了 dict：

```python
df = run_df("fund_etf_category_sina", [{"symbol": "ETF基金"}])
```

`run_df(func, args, kwargs)` 把列表元素按顺序作为**位置参数**拼到 akshare 函数上，
于是 `fund_etf_category_sina(symbol={"symbol": "ETF基金"})` —— `symbol` 收到的是一个 dict。
而 akshare 内部 `fund_map[symbol]` 用该参数作字典 key，dict 不可哈希 → `TypeError`。

真正的错误在我们的调用层：应把 `symbol` 作为**关键字参数**或字符串位置参数传入，
而非把 `{"symbol": ...}` 这个 dict 整体当位置参数。

## 解决办法

将 dict 形式的参数改由 `kwargs` 传递（与 `run_df` 的 `kwargs` 形参语义一致）：

```python
# 修复前
df = run_df("fund_etf_category_sina", [{"symbol": "ETF基金"}])
# 修复后
df = run_df("fund_etf_category_sina", [], {"symbol": "ETF基金"})
```

`_runner.run_df` 在子进程内 `func(*args, **kwargs)` 正确展开，akshare 收到
`fund_etf_category_sina(symbol="ETF基金")` 字符串参数，正常工作。

排查同文件其余 10 处 `run_df` 调用，均为字符串位置参数（如 `fund_overview_em([code])`、
`stock_index_pe_lg([name])` 等），符合各 akshare 函数签名，无同类问题。

## 影响范围

- 仅影响 `-Seed` 启动时的 ETF 基础列表拉取（`AkShareEtfBasicSource.list_etfs`）。
- 修复前：Seed 阶段该源 3 次重试全部失败，基础列表源回落/为空，控制台刷 WARNING，
  且启动时延增加；不影响已落库数据，但首次冷启动无法补全 ETF 主列表。
- 修复后：Seed 正常拉取全部 ETF 列表（实测 1596 只），不再报错。
- 估值/历史分位/成分股等其它 `run_df` 调用链路不受影响（本就没问题）。

## 验证

- `_probe_seed.py` 直接调用 `AkShareEtfBasicSource().list_etfs()`：
  返回 `OK count=1596`，前几行如 `159998 计算机ETF天弘` / `159997 电子ETF天弘` 等，无 TypeError。
- 改动文件 lint 零告警。

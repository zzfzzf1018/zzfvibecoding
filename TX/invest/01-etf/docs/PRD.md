## 1. 产品概述

### 1.1 产品定位
一款面向**中国 A 股市场**的 ETF（交易型开放式指数基金）信息查询工具。用户可通过 **ETF 编号**或 **ETF 名称（支持模糊匹配）** 快速检索目标 ETF，查看其基本信息、跟踪指数的 **PE/PB 估值**、**历史估值分位数**以及**成分股**等核心数据，辅助投资决策。

### 1.2 目标用户
| 用户类型 | 典型场景 | 核心诉求 |
| --- | --- | --- |
| 个人投资者 | 想买某只指数 ETF，先查估值贵不贵 | 快速看 PE/PB、历史分位 |
| 基金/理财顾问 | 给客户筛选低估 ETF | 横向对比估值与分位 |
| 量化/研究员 | 需要成分股、历史估值序列做回测 | 稳定可下载的数据 |
| 开发者/爱好者 | 自建看板或接入自己的系统 | 清晰 API、稳定数据源 |

### 1.3 核心价值
- **省时**：一个入口查全 ETF 关键数据，不用在多个网站间跳转。
- **看得懂**：估值用「历史分位数」直观表达"当前贵不贵"。
- **可追溯**：成分股、估值均可对应到明确的指数与数据源。
- **稳**：多数据源冗余 + 缓存，避免单点失效。

### 1.4 名词解释
| 名词 | 说明 |
| --- | --- |
| ETF | Exchange Traded Fund，交易型开放式指数基金 |
| 跟踪指数 | ETF 所复制/跟踪的标的指数（如沪深300、中证500） |
| PE | 市盈率（Price/Earnings），衡量估值高低 |
| PB | 市净率（Price/Book），衡量净值估值高低 |
| 历史分位数 | 当前 PE/PB 在其自身历史序列中的相对位置（0%~100%），越低代表越"便宜" |
| 成分股 | 跟踪指数所包含的成分股票及其权重 |

## 2. 业务背景与目标

### 2.1 背景
- 中国 ETF 数量已超 1000 只，普通投资者难以快速比较。
- 估值（PE/PB）是判断 ETF 是否低估的核心指标，但多散落在指数公司官网、第三方基金平台，入口分散。
- 历史分位数需要自行拉取序列计算，普通用户无法完成。
- 成分股是判断 ETF 行业分布与风险的关键。

### 2.2 业务目标
1. 提供**统一、稳定**的 ETF 查询入口。
2. 覆盖"检索 → 基本信息 → 估值 → 分位 → 成分股"完整链路。
3. 数据**每日更新**且可溯源，关键指标需标注更新时间。
4. 为后续扩展（自选、对比、提醒、导出）预留接口与数据底座。

### 2.3 成功指标（可量化）
| 指标 | 目标 |
| --- | --- |
| ETF 基础信息覆盖率 | ≥ 95% 在市 A 股 ETF |
| 估值数据可查率 | ≥ 90% 宽基/行业主流 ETF |
| 数据更新延迟 | T+1（交易日次日）完成更新 |
| 检索响应时间（命中缓存） | ≤ 500ms |
| 数据源可用性（月度） | ≥ 99%（含降级） |

## 3. 用户角色与场景

### 3.1 用户角色
- **访客（默认）**：无需登录即可查询。
- **注册用户（远期）**：自选、对比、提醒（本版不实现）。

### 3.2 用户故事
- 作为个人投资者，我想输入"沪深300"或"510300"，看到对应 ETF 列表及估值，以便判断是否买入。
- 作为投资者，我想看到某 ETF 当前 PE 的**历史分位数**，以便判断"现在贵不贵"。
- 作为研究者，我想导出某 ETF 跟踪指数的**成分股及权重**，以便进一步分析。
- 作为开发者，我想通过 API 获取估值与成分股数据，以便接入自有系统。

## 4. 功能需求概览

### 4.1 功能清单
| 编号 | 功能 | 说明 | 优先级 |
| --- | --- | --- | --- |
| F-01 | ETF 检索 | 按编号精确 / 按名称模糊匹配，返回候选列表 | Must |
| F-02 | ETF 基本信息 | 名称、代码、跟踪指数、类型、规模、管理人、费率、成立日等 | Must |
| F-03 | PE/PB 估值 | 当前 PE、PB、对应日期、数据来源 | Must |
| F-04 | 历史分位数 | 当前 PE/PB 在历史序列中的分位、近 N 年区间 | Must |
| F-05 | 成分股 | 跟踪指数成分股列表、权重、行业 | Must |
| F-06 | 历史估值序列 | 时间序列 PE/PB，支持图表与导出 | Should |
| F-07 | 估值对比 | 多只 ETF 估值横向对比 | Could |
| F-08 | 自选/提醒 | 收藏与低估提醒 | Won't（本期） |

### 4.2 优先级方法
采用 MoSCoW：Must（必须有）/ Should（应该有）/ Could（可以有）/ Won't（本期不做）。

## 5. 核心功能详述

### 5.1 F-01 ETF 检索
- **输入**：关键字（编号 6 位数字 或 中文/拼音名称片段）。
- **匹配规则**：
  - 纯数字 → 按代码前缀/精确匹配。
  - 含中文/字母 → 模糊匹配名称、简称、跟踪指数名。
- **输出**：候选 ETF 列表（代码、名称、类型、跟踪指数、最新价），支持分页。
- **体验**：输入即搜（防抖），热门/宽基置顶。

### 5.2 F-02 ETF 基本信息
展示字段：代码、全称、简称、类型（宽基/行业/主题/债券/货币/跨境等）、跟踪指数及代码、基金管理人、托管人、资产规模、份额、成立日期、基金经理、管理费率、托管费率、跟踪误差、场内/场外、交易所。

### 5.3 F-03 PE/PB 估值
- 展示当前 PE（TTM）、PB、股息率（如有）、估值日期。
- 标注数据来源（如中证指数公司 / 第三方）。
- 对无法获取估值的 ETF（如债券、货币、商品类）明确提示"不适用"。

### 5.4 F-04 历史分位数（按月序列）
- 展示形态：**最近 12 个月**、每月一行的 PE 分位与 PB 分位序列（表格），便于观察分位随时间的变化趋势。
- 口径（与主流基金 App 一致，普通用户易懂）：某月分位 = 该月**月末** PE/PB 在「该月末往前 5 年」**滚动窗口**内的历史分位。
- 参数：`months`（默认 12）、`window_years`（默认 5 年）。
- 兜底：上市/数据不足 5 年，用现有全部历史计算并标注"数据不足5年，基准已压缩"；当月无数据的行分位显示"—"。
- 分位说明文案："分位越低代表相对越便宜"。

### 5.5 F-05 成分股
- 列表：股票代码、名称、申万一级行业、权重、所属交易所。
- 支持：按权重排序、按行业筛选、Top N 汇总。
- 展示行业分布（饼图/条形图）。
- 支持导出 CSV。

## 6. 非功能需求
| 类别 | 要求 |
| --- | --- |
| 性能 | 检索命中缓存 ≤ 500ms；冷启动拉取 ≤ 2s |
| 可用性 | 单数据源故障可降级到其他源，不整体不可用 |
| 更新频率 | 基础信息每日增量更新；估值 T+1；成分股按指数公司公告频率 |
| 准确性 | 关键字段标注来源与更新时间，可溯源 |
| 可维护性 | 数据源适配层模块化，新增/替换源成本低 |
| 安全 | 仅只读公开数据；无需用户敏感信息（本期） |
| 合规 | 仅展示公开市场数据，标注来源，不做投资建议 |

## 7. 数据源方案（概览）
详见独立文档《数据源调研与选型》。核心策略：
- **主数据源**：AkShare（Python 聚合库，维护活跃）+ 中证指数公司官方接口。
- **兜底源**：东方财富、新浪财经接口。
- **缓存层**：本地数据库缓存，降低对外请求频率、抗限流。
- **更新**：定时任务每日拉取，失败重试 + 告警。

## 8. 竞品参考（简述）
- 天天基金 / 蛋卷基金 / 且慢：估值展示好，但偏 App、无开放 API、不支持批量。
- 中证指数官网：权威但入口深、不便检索。
- 本工具差异化：统一检索 + 开放数据 + 可扩展。

## 10. 风险与依赖
- 第三方数据源接口变动/限流 → 多源 + 缓存 + 重试。
- 部分 ETF 无指数估值（如债券/货币/商品）→ 明确"不适用"提示。
- 历史数据长度不足 → 分位数计算需最少样本阈值（建议 ≥ 250 个交易日）。
- 合规风险 → 仅展示数据，附免责声明，不做投资建议。

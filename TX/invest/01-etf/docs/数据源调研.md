# 中国股市 ETF 查询工具 — 数据源调研与选型

> 配套文档：PRD、SRS。本文聚焦"稳定数据源"这一关键约束，给出可落地方案与接口示例，供开发直接调用。

---

## 1. 数据需求拆解
ETF 查询的本质是：**ETF 是基金产品，PE/PB 估值与成分股来自其"跟踪指数"**。因此数据分三类：
1. **ETF 基础信息**：代码、名称、管理人、规模、费率、跟踪指数 → 来自基金数据平台。
2. **指数估值（PE/PB）**：指数层面的市盈率/市净率 → 来自指数公司或聚合平台。
3. **指数成分股**：指数成分及权重 → 来自指数公司。

---

## 2. 候选数据源

### 2.1 AkShare（强烈推荐，主数据源）
- 类型：Python 开源库（pip 安装），聚合多家公开接口，维护活跃。
- 优点：一行代码取数、免 token、覆盖广、社区活跃、更新快。
- 缺点：底层依赖第三方网页接口，偶有字段变动（需适配层隔离）。
- 关键接口（示例，版本可能微调，以官方文档为准）：
  ```python
  import akshare as ak
  # ETF 列表（新浪）
  ak.fund_etf_category_sina(symbol="ETF基金")
  # ETF 实时行情（东方财富）
  ak.fund_etf_spot_em()
  # 指数成分股（中证）
  ak.index_stock_cons_csindex(symbol="000300")   # 沪深300
  # 指数估值（东方财富）
  ak.stock_index_value_name_em()                 # 指数估值总表
  # 个股估值（用于回算/校验）
  ak.stock_a_indicator_lg(symbol="000001", date="20260101")
  ```
- 注意：AkShare 接口名随版本演进，开发前先 `pip install akshare --upgrade` 并核对函数签名。

### 2.2 中证指数公司官网（权威源，估值/成分股兜底）
- 官网：https://www.csindex.com.cn
- 优点：最权威，PE/PB 与成分股官方发布。
- 缺点：以网页/JSON 接口为主，需解析；更新频率随公告。
- 关键接口（研究用，需自行抓包确认）：
  - 估值：`https://www.csindex.com.cn/csindex-home/valuation/indices`（POST，参数含指数代码）
  - 成分股：`https://www.csindex.com.cn/csindex-home/component/detail`（成分股明细）
- 用法：作为 AkShare 估值/成分股结果的**交叉校验与兜底**。

### 2.3 东方财富（东方财富，行情/基础信息兜底）
- 接口：fund.eastmoney.com 系列（AkShare 已封装 `fund_etf_spot_em` 等）。
- 优点：行情更新快、稳定。
- 用法：实时行情、ETF 列表兜底。

### 2.4 新浪财经（基础信息兜底）
- 接口：AkShare 已封装 `fund_etf_category_sina`。
- 用法：ETF 列表兜底。

### 2.5 Tushare（备选，需 token）
- 类型：注册获取 token，有积分限制；数据规范、稳定。
- 优点：数据质量高、有 SLA 感。
- 缺点：需注册、免费额度有限、指数估值接口覆盖有限。
- 用法：作为 AkShare 失效时的二级兜底（尤其中证/国证指数元数据）。

### 2.6 第三方估值展示（蛋卷/且慢/中证估值）
- 蛋卷基金、且慢提供指数估值展示，但偏前端、无开放 API，仅作**人工核对参考**，不建议做程序依赖。

## 3. 推荐数据源组合（落地方案）

| 数据 | 主源 | 兜底源 | 缓存 |
| --- | --- | --- | --- |
| ETF 基础信息 | AkShare(sina/em) | 东方财富直连 | 24h |
| 实时行情 | AkShare(em) | 新浪 | 日内 |
| 指数估值 PE/PB | AkShare(估值表) + 中证官网 | Tushare(部分) | T+1 |
| 指数成分股 | AkShare(中证) | 中证官网直连 | 7d/公告触发 |
| 历史估值序列 | 自采：每日落库 `index_valuation` | — | 自管 |

**核心策略**：
1. **主源失败自动降级**：适配层依次尝试 主 → 兜底。
2. **自采历史序列**：估值历史无法稳定直接拉全量，建议从"上线日起每日落库最新估值"，逐步积累历史；对存量历史可用 AkShare/中证历史接口补录。
3. **缓存抗限流**：所有外部请求先查缓存，未命中再请求，显著降低被封风险。

---

## 4. 历史分位数计算方案
- 数据：自管 `index_valuation_history`（每日 PE/PB）。
- 算法：
  ```
  percentile = count(历史值 <= 当前值) / count(全部历史值) * 100
  ```
- 最小样本：≥ 250 交易日（约 1 年），不足则窗口降级并提示。
- 周期支持：1y / 3y / 5y / 全历史（按日期过滤后计算）。

---

## 5. 稳定性与运维设计
- **超时/重试**：外部请求超时 10s，失败指数退避重试 ≤ 3 次。
- **限流保护**：单进程请求频率限制（如 AkShare 接口间隔 ≥ 0.3s）。
- **失败告警**：连续失败 N 次写日志 + 通知（邮件/Webhook）。
- **数据新鲜度监控**：记录每个源的 `update_time`，超期标"数据可能延迟"。
- **适配层隔离**：所有外部调用只在 `datasource/` 模块内，业务层不直接依赖 AkShare，便于替换。

## 6. 环境与依赖（开发用）
```text
Python >= 3.10
akshare >= 1.13
fastapi, uvicorn        # API 服务
sqlalchemy + sqlite     # 缓存/存储（MVP）；可换 PostgreSQL
schedule / apscheduler  # 定时更新
requests, pandas        # 直连解析
streamlit (可选)        # MVP 快速验证 UI
```
安装：
```bash
pip install akshare fastapi uvicorn sqlalchemy pandas requests apscheduler
# 可选 MVP
pip install streamlit
```

---

## 7. 风险与缓解
| 风险 | 缓解 |
| --- | --- |
| AkShare 接口字段变动 | 适配层隔离 + 单测 + 监控告警 |
| 中证官网反爬/接口改版 | 多源兜底 + 缓存 |
| 部分 ETF 无指数估值（债券/货币/商品） | 显式 `applicable:false` |
| 历史序列不足 | 上线起自采 + 补录存量 |
| 网络出口受限 | 部署在有外网机器；或定期离线导入 |

---

## 8. 给接手 AI 的实操建议
1. 先 `pip install akshare` 跑通 `fund_etf_category_sina` 与 `index_stock_cons_csindex`，确认环境可用。
2. 建 `datasource/` 包，按 SRS 的 IR-01 定义抽象，先实现 AkShare 适配器。
3. 建 `models.py`（SQLAlchemy）对应 DR-01~05。
4. 写 `scheduler` 每日落库估值，逐步积累历史。
5. 用 Streamlit 出 MVP 验证全链路，再上 FastAPI + 前端。
6. 所有外部源失败时返回结构化错误，确保 UI 不崩。

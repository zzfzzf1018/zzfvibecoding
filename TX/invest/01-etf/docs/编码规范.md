# 编码规范（Coding Standards）

| 项 | 内容 |
| --- | --- |
| 版本 | v1.0 |
| 日期 | 2026-07-10 |
| 语言 | Python ≥ 3.10 |
| 目标 | 可读性、一致性、松耦合、可测试 |

---

## 1. 代码风格
- 遵循 **PEP 8**；用 `ruff`/`black` 自动格式化（行宽 100）。
- `isort` 管理 import 顺序（标准库 → 第三方 → 本地，分组）。
- 禁止裸 `print` 调试；统一 `core/logging` 的 `logger`。
- 禁止魔法数字/字符串：常量集中定义（如 `MIN_SAMPLES=250`、`WINDOWS`）。

---

## 2. 类型与注解
- 所有函数/方法必须带类型注解（参数 + 返回）。
- 优先 `typing.Protocol` 定义端口；具体类实现端口。
- 出参使用 `pydantic` 模型或 `dataclass`（见 `models/schemas.py`），禁止直接返回 ORM 对象出 services 层。

---

## 3. 命名约定
| 对象 | 规则 | 示例 |
| --- | --- | --- |
| 模块/包 | 小写+下划线 | `valuation_service.py` |
| 类 | 大驼峰 | `ValuationService` |
| 函数/变量 | 小写+下划线 | `get_latest` |
| 常量 | 大写下划线 | `MIN_SAMPLES` |
| 抽象端口 | XxxSource / XxxRepository | `ValuationSource` |
| 异常 | XxxError | `SourceUnavailable` |

---

## 4. 模块与耦合（硬性）
- **分层**：`api / services / datasource / repositories / models / core`，单向依赖。
- `services/` **严禁** `import akshare` 或 `from app.datasource.akshare_src import ...`；只 `from app.datasource.interfaces import ...`。
- 外部请求/解析只在 `datasource/*`、`core/clients/*`。
- SQL 只在 `repositories/`。
- 配置走 `core/config`，不硬编码。
- 依赖通过 `core/di.py` 注入；不要在各层自行 `new` 具体源。

---

## 5. 函数设计
- 单一职责：一个函数只做一件事。
- 小函数优先（建议 ≤ 40 行）；复杂逻辑拆子函数。
- 纯函数优先（分位计算、聚合），便于单测。
- 不吞异常：捕获后记录日志并向上抛 `AppError` 子类或返回 None（由调用方降级）。

---

## 6. 错误处理
- 统一异常基类 `AppError`（见详细设计 §5），带 `code/message`。
- 外部调用必须 try/except，返回 None 或抛语义化异常；不向用户暴露堆栈。
- 参数校验前置，非法即 `InvalidParam(400)`。

---

## 7. 日志与可观测
- 用 `core/logging` 的结构化 logger；关键路径记录 `index_code`/`source`/`cost_ms`。
- 禁止记录密钥/敏感信息。
- 数据源失败记录 `source` 与原因，供告警。

---

## 8. 配置与 secrets
- 配置项经 `core/config`（pydantic-settings）读环境变量 / `.env`。
- `.env` 不入库（已在 `.gitignore`）。
- 绝不硬编码 token/密码；新增配置需在 config 与 README 注明。

---

## 9. 测试
- 框架 `pytest`；单测不依赖外网（用 Mock 实现 `interfaces`）。
- 命名 `test_*.py`，函数 `test_xxx`；覆盖：分位、模糊检索、源降级、参数校验。
- 必须包含**松耦合护栏测试**：静态断言 `services/` 不 import 具体数据源。
- 测试数据用 fixtures，不污染生产库（测试用临时 SQLite）。

---

## 10. 依赖管理
- 依赖写入 `requirements.txt`（或 `pyproject.toml`），注明用途。
- 仅安装必要包；禁止全局随意 `pip install`。
- 锁定主要版本，保证可重现。

---

## 11. 注释与文档
- 公共类/函数写 docstring（目的、参数、返回、异常）。
- 复杂算法（分位、降级）加简要注释说明意图。
- 代码与 `SRS/设计文档` 保持一致；接口变更同步文档。

---

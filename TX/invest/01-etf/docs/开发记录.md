## CHG-20260710-05
- 改动人：AI（CodeBuddy）
- 日期：2026-07-10
- 关联需求：一键启动"没反应"修复；README 使用文档补全
- 改动目的：用户双击一键启动脚本后无任何反应。根因有二：
  1) Windows 默认 PowerShell 执行策略为 Restricted，直接双击 `.ps1` 被静默拦截；
  2) `scripts/run.ps1` 含中文（UTF-8 无 BOM），Windows PowerShell 5.1 按系统 GBK 读取，
     中文字符第二字节恰为 `"`(0x22) 导致字符串被提前关闭，引发整段解析级联错误（脚本在解析阶段即整体中止，故"没反应"）。
- 涉及文件：
  - run.bat（新增）：Windows 双击启动器，`powershell -ExecutionPolicy Bypass -NoExit -File scripts\run.ps1`，绕过执行策略且窗口不消失
  - scripts/run.ps1（重写）：内容改为纯英文（注释/提示），规避 GBK 解析级联错误；uvicorn 改为前台运行（同窗口日志）；浏览器打开用 `Start-Job` 传参避免 `$Port` 作用域丢失；`-Seed` 用临时 .py 文件+环境变量 `SEED_SRC`，避免 here-string 引号转义坑
  - run.py（微调）：`main()` 加 try/except，Windows 双击报错时停于"按 Enter 退出"防止一闪而过
  - README.md（扩充）："一键运行（推荐）"新增三种启动方式、如何打开网页、`没反应/排错` 小节
- 改动摘要（before → after）：
  - before：`run.ps1` 中文+here-string，双击被执行策略拦截且自身解析报错；uvicorn 用 `Start-Process` 开在另一窗口
  - after：`run.bat` 双击即绕过策略前台运行；`run.ps1` 纯英文可解析；uvicorn 前台同窗口、日志可见
- 接口/契约影响：无（仅运行/文档，未改业务代码）
- 测试情况：真实启动验证（端口 8022）——
  - `GET /health` → `{"status":"ok"}`；`GET /` → `<!doctype html>`（Web UI 正常托管）；前台进程持续运行直到被关闭，证明启动链路已通
- 风险/回滚：纯启动脚本/文档改动，零业务风险；如需回退，删除 run.bat、将 run.ps1 还原为旧版即可

- 算法（方案 B）：某月分位 = 该月「月末」（当月最后一条有效记录）PE/PB 在「该月末往前 `window_years` 年」滚动窗口内序列的 `percentile_rank`；当月无数据 → 分位 None；可用历史不足 `window_years` 年 → 全历史兜底并置 `degraded=true`；无任何历史 → 空 `series`（不抛错，属正常业务）。

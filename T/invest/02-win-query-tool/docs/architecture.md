## 模块划分

### 1. Electron主进程模块

**文件**: `src/main/index.js`

**职责**:
- 窗口创建和管理
- Python子进程启动和监控
- 服务健康检查轮询
- IPC通信处理
- 文件对话框和下载管理

**核心流程**:

```
app.whenReady()
    │
    ├── 设置userData路径（修复沙箱问题）
    │
    ├── 创建窗口（隐藏状态）
    │
    ├── 启动Python服务
    │
    ├── 轮询 /api/health 等待就绪
    │       │
    │       ├── 成功 → 显示窗口
    │       │
    │       └── 超时 → 显示错误并退出
    │
    └── 监听窗口事件
            │
            ├── window-all-closed → 关闭Python进程，退出应用
            │
            └── activate → 重新创建窗口
```

### 2. Flask API服务模块

**文件**: `src/python/app.py`

**职责**:
- RESTful API路由定义
- akshare数据获取和处理
- 缓存逻辑集成
- 错误处理和日志记录

**API路由列表**:

| 路由 | 方法 | 功能 | 处理函数 |
|------|------|------|----------|
| /api/health | GET | 健康检查 | health() |
| /api/cache/stats | GET | 缓存统计 | cache_stats() |
| /api/cache/clear | POST | 清除缓存 | clear_cache() |
| /api/cache/clear_expired | POST | 清除过期缓存 | clear_expired_cache() |
| /api/stock/search | GET | 股票搜索 | search_stock() |
| /api/stock/finance_report | GET | 财务报表 | finance_report() |
| /api/stock/analysis | GET | 公司分析 | stock_analysis() |
| /api/stock/download_report | GET | 下载报表 | download_report() |
| /api/stock/prospectus | GET | 招股书查询 | prospectus() |
| /api/stock/download_prospectus | GET | 下载招股书 | download_prospectus() |

### 3. 缓存模块

**文件**: `src/python/cache.py`

**职责**:
- SQLite数据库初始化和管理
- 缓存数据的存储和读取
- 缓存过期策略管理
- 缓存统计信息

**数据库结构**:

```sql
CREATE TABLE cache (
    key TEXT PRIMARY KEY,      -- 缓存键（MD5哈希）
    data TEXT NOT NULL,        -- 缓存数据（JSON字符串）
    created_at TIMESTAMP,      -- 创建时间
    updated_at TIMESTAMP       -- 更新时间
);
```

**核心函数**:

| 函数名 | 功能 | 参数 | 返回值 |
|--------|------|------|--------|
| init_cache() | 初始化数据库表 | 无 | 无 |
| get_cache_key() | 生成缓存键 | prefix, params | string |
| set_cache() | 设置缓存 | prefix, params, data | 无 |
| get_cache() | 获取缓存 | prefix, params | data/null |
| clear_cache() | 清除所有缓存 | 无 | 无 |
| clear_expired_cache() | 清除过期缓存 | 无 | 无 |
| get_cache_stats() | 获取缓存统计 | 无 | stats object |

## 数据流程

### 1. 股票搜索流程

```
用户输入关键词 → 前端请求 → Flask路由 → akshare获取数据 → 缓存写入 → 返回结果
                                                   │
                                                   ▼
                                              缓存读取（网络失败时）
```

### 2. 财务报表流程

```
选择股票 → 前端请求 → Flask路由 → akshare获取报表 → 数据格式转换 → 缓存写入 → 返回结果
                                                          │
                                                          ▼
                                                     缓存读取（网络失败时）
```

### 3. 报表下载流程

```
点击下载 → 前端请求 → Flask路由 → akshare获取数据 → 生成Excel文件 → 返回文件路径 → 前端文件对话框 → 复制文件到目标位置
```

### 4. 缓存流程

```
请求到达
    │
    ├── 检查缓存键是否存在
    │       │
    │       ├── 存在且未过期 → 返回缓存数据（cached: true）
    │       │
    │       └── 不存在或已过期 → 调用akshare
    │               │
    │               ├── 成功 → 更新缓存 → 返回新数据（cached: false）
    │               │
    │               └── 失败 → 返回错误（无缓存时）或缓存数据（有缓存时）
```

## 关键设计决策

### 1. 技术栈选择

| 层级 | 技术 | 理由 |
|------|------|------|
| 桌面框架 | Electron | 跨平台支持，HTML/CSS/JS开发，成熟稳定 |
| 前端框架 | React | 组件化开发，虚拟DOM，生态丰富 |
| UI组件 | Ant Design | 组件丰富，设计规范统一 |
| 数据服务 | Flask | 轻量级，易于开发，与Python生态兼容 |
| 数据源 | akshare | 国内最全金融数据源，支持A股/港股 |
| 缓存 | SQLite | 轻量级嵌入式数据库，无需额外服务 |

### 2. 进程通信设计

- **Electron ↔ Flask**: HTTP RESTful API
- **Electron主进程 ↔ 渲染进程**: IPC通信
- **Flask ↔ akshare**: Python函数调用

### 3. 错误处理设计

- **网络错误**: 返回明确错误信息，有缓存时返回缓存数据
- **akshare错误**: 返回错误信息，记录日志
- **参数错误**: 返回错误信息，提示正确格式
- **服务启动失败**: 显示错误对话框，引导用户检查环境

### 4. 缓存策略设计

- **缓存有效期**: 24小时（财务数据变化不频繁）
- **缓存键设计**: MD5(接口名 + JSON参数)，确保唯一性
- **自动清理**: 服务启动时清理过期缓存
- **手动管理**: 提供缓存统计和清除接口

## 部署说明

### 开发环境

```bash
# 安装依赖
pip install -r src/python/requirements.txt
cd src/renderer && npm install

# 构建前端
cd src/renderer && npm run build

# 启动服务
python src/python/app.py

# 启动Electron
npm start
```

### 生产环境

```bash
# 打包应用
npm run build

# 生成安装包
electron-builder
```

## 安全考虑

- 输入验证：对所有用户输入进行验证
- 错误信息：不暴露敏感信息
- 文件路径：使用绝对路径，避免路径遍历攻击
- 网络请求：使用HTTPS（生产环境）
- 权限控制：最小化权限原则
